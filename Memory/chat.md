# Chat Export

Дата выгрузки: 2026-06-13

## Кратко

Этот файл фиксирует содержание текущего рабочего чата по проекту каталога ресурсов `Lkit`: наполнение данных, сбор weekly-кейсов, изменения структуры рубрик, переносы карточек, удаление записей и настройку навыка `шерсти`.

## Хронология

### 1. Framer / Obsidian / импорт

- Обсуждалось наполнение Framer-шаблона контентом из Obsidian.
- Был сделан локальный навык и конвертер `Obsidian -> Framer CMS`.
- Настраивалась браузерная автоматизация через Edge/CDP.
- Каталог из Obsidian был разобран и подготовлен к импорту.
- В проекте Framer были расширены категории и проведён массовый импорт карточек.
- Частично были сняты скриншоты сайтов и загружены превью во Framer.

### 2. Сбор локального каталога

- Была собрана единая таблица ресурсов.
- Выполнялись:
  - дедупликация,
  - очистка мёртвых сервисов,
  - классификация по рубрикам и подрубрикам,
  - enrichment описаниями,
  - добавление статусов доступа,
  - сбор скриншотов и ссылок на них.
- На базе таблицы был собран отдельный локальный сайт каталога.

### 3. Структура сайта

- Создан сайт `Lkit` в тёмной теме.
- Слева размещён вертикальный рубрикатор.
- Поиск сделан глобальным по каталогу.
- Под строкой поиска появились табы-фильтры.
- Карточки стали компактными: заголовок, краткое описание, эмодзи/обложка.
- Затем был добавлен раздел `Викли`.

### 4. Большие изменения по рубрикам

Были закреплены и многократно правились рубрики:

- `Инструменты`
- `ИИ`
- `Ресурсы`
- `Продвижение`
- `Контент и обучение`

Отдельная рубрика `Доступность` позже была убрана и перенесена в:

- `Контент и обучение -> Доступность`

Раздел `Викли` был добавлен первым в порядке рубрик.

### 5. Правки данных каталога

В чате многократно переносились и удалялись карточки.

Примеры действий:

- курс `Google UX Design Professional Certificate` и `Foundations of User Experience (UX) Design` были добавлены в `Контент и обучение -> Курсы и школы`;
- `Deceptive Design Types` был добавлен в `Контент и обучение -> Медиа и блоги`;

### 8. Weekly / Викли

Сначала weekly был сделан неправильно:

- туда попадали поисковые ссылки;
- превью были битые или просто screenshots сайтов.

Затем weekly был перестроен:

- раздел `Викли` оставлен как отдельный контейнер;
- в него начали попадать только прямые кейсы;
- карточки weekly стали показывать:
  - обложку,
  - заголовок,
  - автора,
  - категорию,
  - источник.

Были добавлены weekly-фильтры:

- `UX/UI`
- `Брендинг`

Сама выдача weekly стала перемешиваться между категориями.

### 9. Weekly collector

Был создан weekly collector:

- `scripts/collect_weekly_cases.mjs`

Он строился как pipeline:

- browser/CDP + Playwright,
- прямые ссылки на кейсы,
- обложки,
- structured data,
- OpenAI API для отбора,
- fallback-режим без падения сборки.

Отдельно:

- `Dribbble` был доведён до рабочего состояния;
- затем был добавлен `Behance`;
- для `Behance` стали извлекаться:
  - прямые `gallery` URL,
  - `og:image`,
  - author,
  - likes/views из `ld+json`.

### 10. Навык `шерсти`

Был создан и установлен навык:

- `skills/shersti/SKILL.md`
- `skills/shersti/agents/openai.yaml`
- установленная копия в `~/.codex/skills/shersti`

Смысл навыка:

- запускать weekly-сборщик по слову `шерсти`
- пересобирать `weekly_cases.json`
- пересобирать сайт

Позже навык был уточнён:

- по умолчанию `шерсти` означает последовательный сбор:
  1. `Dribbble`
  2. `Behance`

Также были добавлены режимы:

- `шерсти dribbble`
- `шерсти behance`
- `шерсти all`

### 11. Финальная weekly-квота

Итоговое требование было доведено до:

- показывать в weekly и `Dribbble`, и `Behance`
- фиксировать квоты по категориям
- затем увеличить объём до:
  - `20` материалов с `Dribbble`
  - `20` материалов с `Behance`

На момент последнего шага логика сборщика была доведена до модели:

- mixed weekly
- по обоим источникам
- с квотами по категориям
- с прямыми ссылками и реальными cover image URL
