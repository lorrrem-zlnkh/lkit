# Shеrsti

Используй этот навык, когда пользователь просит `шерсти`, `обнови викли`, `собери weekly-кейсы` или просит подборку кейсов из `Dribbble` и `Behance`.

По умолчанию `шерсти` означает последовательный сбор из обоих источников:
1. сначала `Dribbble`
2. затем `Behance`

Не ограничивайся одним источником, если пользователь явно не попросил `шерсти dribbble` или `шерсти behance`.

## Что делает навык

1. Проверяет, что доступны:
   - `OPENAI_API_KEY`
   - браузерная сессия через `CDP` на `http://127.0.0.1:9222`
2. Запускает weekly-сборщик в последовательном режиме `Dribbble -> Behance`:
   - `npm run weekly:collect`
3. Пересобирает данные сайта:
   - `npm run catalog:build-data`
4. Проверяет результат в:
   - `output/weekly_cases.json`

## Обязательные переменные

Используй такие env vars при запуске:

```bash
OPENAI_API_KEY=...
WEEKLY_USE_CDP=1
WEEKLY_CDP_URL=http://127.0.0.1:9222
WEEKLY_SOURCES=all
WEEKLY_LIMIT=40
WEEKLY_CASE_LIMIT_PER_SOURCE=30
```

`WEEKLY_SOURCES=all` означает последовательный сбор из обоих источников.
`WEEKLY_CASE_LIMIT_PER_SOURCE=30` означает минимум 30 кандидатов с каждого источника до этапа AI-отбора.

## Команды

Основной запуск:

```bash
OPENAI_API_KEY=... WEEKLY_USE_CDP=1 WEEKLY_CDP_URL=http://127.0.0.1:9222 WEEKLY_SOURCES=all WEEKLY_LIMIT=40 WEEKLY_CASE_LIMIT_PER_SOURCE=30 npm run weekly:collect
```

Затем:

```bash
npm run catalog:build-data
```

## Ограничения

- Не публикуй карточки с поисковыми URL, feed-страницами или пустыми картинками.
- Оставляй только прямые ссылки на кейсы и реальные image URLs.
- Если `Behance` не отдаёт валидные кейсы, зафиксируй это явно и не подмешивай мусорные ссылки.
- Если `OpenAI API` не вернул отбор, используй fallback только из валидных direct-case кандидатов.

## Что сообщать пользователю

- сколько кандидатов собрано с `Dribbble`
- сколько кандидатов собрано с `Behance`
- сколько кейсов попало в `weekly_cases.json`
- какие категории реально попали в weekly (`UX/UI`, `Брендинг`)
- если один из источников не отработал, сказать это прямо
